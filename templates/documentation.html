CronPulse — User Guide

Simple, reliable monitoring for scheduled jobs. This guide explains the user-facing workflows — signing up, creating monitors, managing alerts, and handling API keys.

Getting started

  1. Create an account using the Sign Up page.
  2. Log in with your email and password.
  3. Visit the Dashboard to see your monitors and account information.

Dashboard overview

The Dashboard shows a summary of your monitors, their status, last ping time, and next expected ping. Use the navigation to jump to full monitor management or API key management.

Creating a monitor

To add a monitor, provide:

  • Name — friendly label for the monitor.
  • Interval — how often the job should run (minutes).
  • Alert destination — at least one of an email address or a webhook URL where alerts will be sent.
  • Expiration (optional) — an optional date/time when the monitor should stop generating alerts.

After creation the monitor appears in your list. You can edit or delete it from the UI.

Editing & deleting

Edit a monitor to change interval or alert settings. Delete a monitor to stop monitoring permanently — deletions cannot be undone.

Alerts & notifications

When a monitor misses its expected ping, CronPulse will send alerts to the configured destinations. Ensure your email filters allow messages from the service and that webhook endpoints accept POST requests.

API keys

If you need programmatic access, create an API key from the API Keys page. Important notes:

  • When a key is created the secret is shown once — copy and store it securely. It cannot be retrieved later.
  • Treat API keys like passwords. Do not share them publicly or check them into source control.
  • Rotate (delete and recreate) keys if you suspect they were exposed.

Best practices

  • Set alert destinations you monitor (email you check, or a webhook that your systems handle).
  • Choose conservative intervals (don't set overly frequent checks unless needed).
  • Keep API keys in a secure secrets manager or environment variable; rotate periodically.

Troubleshooting

  • Can't log in: Try resetting your password from the login page. If problems persist, contact support.
  • No alert emails: Check spam/junk folders and that the email address is correct.
  • Webhook failures: Confirm your endpoint returns a 200/2xx response and is reachable from the service.
  • Created API key not visible: The secret is shown only once. If lost, delete and recreate the key.

Contact & support

For help, contact the service administrator or open a support request (details available in your account settings).
